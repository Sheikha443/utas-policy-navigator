# 🗄️ كيف أربط النظام بقاعدة بيانات حقيقية؟ (خطوة بخطوة · بالعربي)

> اقرأ هذا الملف مرّة واحدة وستفهم كل شيء. لا يحتاج خبرة برمجية.
> الوقت المتوقّع: **٥ إلى ١٠ دقائق**.

---

## أولاً: ما الفرق بين «الوضع التجريبي» و«قاعدة البيانات الحقيقية»؟

| | الوضع التجريبي (الآن) | قاعدة البيانات الحقيقية (بعد هذه الخطوات) |
|---|---|---|
| أين تُحفظ الحسابات؟ | داخل متصفح **كل جهاز على حدة** | في **مكان مركزي واحد** على الإنترنت |
| هل يراها الجميع؟ | لا — كل جهاز منفصل | **نعم** — أي طالب من أي جهاز |
| بريد التفعيل | رمز يظهر داخل الشاشة | **رسالة حقيقية تصل لبريد الطالب** |
| يحتاج إعداد؟ | لا، يعمل فوراً | نعم، الخطوات بالأسفل |

النظام **مبرمَج بذكاء**: بمجرّد أن تضع المفاتيح في الخطوة ٥، يتحوّل **تلقائياً** إلى الوضع الحقيقي. وإن تركتها فارغة، يبقى في الوضع التجريبي. لا تحتاج تعديل أي شيء آخر.

---

## الخطوات

### 1) أنشئ مشروعاً في Firebase (مجّاني)
1. افتح: **https://console.firebase.google.com**
2. سجّل الدخول بحساب Google.
3. اضغط **«Add project / إضافة مشروع»**، اكتب اسماً (مثل `utas-integrity`)، ثم **Continue** حتى ينتهي. (يمكنك تعطيل Google Analytics.)

### 2) فعّل تسجيل الدخول بالبريد
1. من القائمة الجانبية: **Build → Authentication**.
2. اضغط **Get started**.
3. اختر **Email/Password**، فعّله (Enable)، ثم **Save**.

### 3) أنشئ قاعدة البيانات (Firestore)
1. من القائمة: **Build → Firestore Database**.
2. اضغط **Create database**.
3. اختر **Start in production mode**، ثم اختر الموقع (مثلاً `eur3` أو الأقرب)، ثم **Enable**.
4. بعد إنشائها، افتح تبويب **Rules** وألصق هذا، ثم **Publish**:

```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /users/{uid} {
      allow read, write: if request.auth != null && request.auth.uid == uid;
      match /subs/{doc} {
        allow read, write: if request.auth != null && request.auth.uid == uid;
      }
    }
  }
}
```
> هذا يضمن أن كل طالب يرى بياناته فقط.

### 4) احصل على المفاتيح
1. اضغط أيقونة الترس ⚙️ (أعلى اليسار) → **Project settings**.
2. انزل إلى **Your apps** واضغط أيقونة الويب **`</>`**.
3. اكتب اسماً (مثل `web`) واضغط **Register app**.
4. ستظهر لك قطعة كود فيها `firebaseConfig = { ... }`. **هذه هي المفاتيح.**

### 5) ضع المفاتيح في النظام
1. افتح الملف: **`js/firebase-config.js`**
2. انسخ القيم من Firebase إلى أماكنها. سيصبح هكذا (بقيمك أنت):

```js
window.FIREBASE_CONFIG = {
  apiKey:            "AIzaSyXXXXXXXXXXXXXXXXX",
  authDomain:        "utas-integrity.firebaseapp.com",
  projectId:         "utas-integrity",
  storageBucket:     "utas-integrity.appspot.com",
  messagingSenderId: "1234567890",
  appId:             "1:1234567890:web:abcdef123456",
};
```

3. احفظ الملف.

### 6) اسمح بنطاق موقعك (مهم)
في Firebase: **Authentication → Settings → Authorized domains** → أضِف نطاق موقعك على GitHub
(مثال: `username.github.io`). للتجربة محلياً `localhost` مضاف تلقائياً.

---

## ✅ تمّ! ماذا يحدث الآن؟
- ارفع المشروع على GitHub كالمعتاد.
- عند فتح بوابة الطالب ستجد الشارة تحوّلت إلى: **«متصل بقاعدة البيانات السحابية»** 🟢
- أي طالب يسجّل → يصله **بريد تفعيل حقيقي** → بياناته تُحفظ مركزياً → تراها كلها في Firestore.

---

## 🔁 طريقة التفعيل في الوضع الحقيقي
في الوضع السحابي، التفعيل يكون **برابط في البريد** (وليس رمزاً):
1. يسجّل الطالب → تصله رسالة فيها رابط.
2. يضغط الرابط.
3. يرجع للبوابة ويضغط **«لقد فعّلت — تابِع»**.

(الشاشة تشرح ذلك تلقائياً — لا تحتاج فعل شيء.)

---

## أسئلة شائعة

**هل المفاتيح سرّية؟** لا. مفاتيح Firebase للويب عامة بطبيعتها، والحماية تتم عبر «Rules» التي وضعناها في الخطوة ٣.

**هل أحتاج بطاقة بنكية؟** لا. الخطة المجّانية (Spark) تكفي لمشروع جامعي.

**لو واجهت مشكلة؟** تأكّد من: (أ) تفعيل Email/Password، (ب) نشر الـ Rules، (ج) إضافة نطاق موقعك في Authorized domains.
